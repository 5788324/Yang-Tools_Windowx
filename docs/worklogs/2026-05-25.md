## 已完成

- 提交并推送上一轮插件管理中心改动：
  - 提交：`166f276 feat: add managed plugin install workflow`
  - 远端：`main -> origin/main`
- 新增插件权限摘要字段：
  - `RawPluginManifest.permissions`
  - `PluginSummary.permissions`
- 在插件扫描阶段合并和推断权限：
  - manifest 显式 `permissions`
  - `files` 触发推断 `file-read`、`file-write`
  - `img` 触发推断 `image-read`
  - manifest 文本中出现 clipboard/screenshot/capture/translate/http/openexternal 时推断相关权限
- 主界面插件标签区域新增权限标签展示。
- 插件运行上下文新增 `permissions`，插件窗口 preload 可通过兼容 API 查询：
  - `getGrantedPermissions()`
  - `hasPermission(permission)`
- 主界面新增插件运行前确认弹窗：
  - 插件列表“打开”会先确认权限
  - 搜索“运行匹配”也会先确认权限
  - 确认后才真正打开插件窗口
- 新增插件信任状态：
  - `src/main/pluginTrustStore.ts`
  - 信任保存到 `{userData}/plugin-trust.json`
  - 信任指纹包含来源、ID、版本、权限集合
  - 弹窗默认勾选“记住此版本和权限组合”
  - 同一插件版本和权限集合再次打开时跳过确认
- 新增兼容 API 权限拦截：
  - 剪贴板文本读写需要 `clipboard`
  - 剪贴板历史搜索需要 `clipboard-history`
  - 通知需要 `notification`
  - 外部链接需要 `shell-open`
  - AI 工具注册需要 `ai-tools`
  - 插件本地 DB 需要 `db`
- 权限推断从 manifest 扩展到本地插件源码关键字，覆盖 `copyText`、`db`、`registerTool` 等常见用法。
- 生成免安装测试包：
  - `release/Yang-Tools-0.1.0-portable`
  - `release/Yang-Tools-0.1.0-portable.zip`
  - 包内包含 Electron 运行时、`out/` 构建产物、最小运行依赖、`resources/local-plugin-library`
  - 包内 README：`README-使用说明.txt`
- 修复用户打开计算稿纸时报错 `An object could not be cloned.`：
  - 原因是 Vue 响应式对象/数组被直接传给 Electron IPC
  - `trustGrant(plugin)` 现在会复制 `permissions`
  - `openSamplePlugin` 前会把请求重新组装为纯 `OpenPluginRequest`
  - 修复后重新生成 portable 包

## 问题与风险

- 权限目前已用于界面展示和运行前确认。
- 兼容桥已对剪贴板、通知、外部链接、AI 工具注册和 DB 做权限拦截。
- 信任状态还没有做设置页管理入口，只能通过文件或后续功能撤销。
- 权限推断是保守辅助信息，不能代替真正的代码审计和沙箱策略。
- 免安装包内包含本机插件样本，只适合本机测试，不适合公开分发。
